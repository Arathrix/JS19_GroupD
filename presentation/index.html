Document Object Model

Programming interface for representation and interaction with objects in HTML, XHTML and XML documents

Is the HTML you write the DOM? No.
Is 'View Source' the DOM? No.

DOM hierarchy

Nodes are organized in a tree structure -> DOM tree

The document hierarchy provides us with an 'address' for each element.

< !DOCTYPE html >

The doctype is not actually a tag, but a declaration, telling the browser what kind of HTML we are using. In this case: HTML5.

< html > < /html >

Defines the boundaries of the HTML document.

< head > < /head >

The < head > element is a container for metadata.

document title, styles, links, scripts and other

Metadata is not displayed.

< body > < /body >

The < body > element contains all of the contents for an HTML document.

text, hyperlinks, images, tables, lists..

LESS

@base: #888888;

.box {
  color: saturate(@base, 5%);
  border-color: lighten(@base, 30%);
  div { box-shadow: 10px 10px 5px @base; }
}

compiles to:

.box {
  color: #fe33ac;
  border-color: #fdcdea;
}
.box div {
  box-shadow: 10px 10px 5px #888888;
}

Mixins in LESS

.a, #b {
  color: red;
}
.mixin-class {
  .a();
}
.mixin-id {
  #b();
}

compiles to:

.a, #b {
  color: red;
}
.mixin-class {
  color: red;
}
.mixin-id {
  color: red;
}

Mixins II

.my-hover-mixin() {
  &:hover {
    border: 1px solid red;
  }
}
button {
  .my-hover-mixin();
}

compiles to:

button:hover {
  border: 1px solid red;
}
